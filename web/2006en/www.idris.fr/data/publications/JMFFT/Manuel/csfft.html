LAST NAME

     SCFFT, CSFFT - Apply Fast Fourier Transform (FFT)
                    real-complex or complex-real.

SYNTAX

     CALL SCFFT (isign, n, scale, x, y, table, work, isys)

     CALL CSFFT (isign, n, scale, x, y, table, work, isys)

IMPLEMENTATION

     These sub-programs emulate the sub-programs of the same name of the SCILIB of
     CRAY. All real or complex arguments must be declared
     in double precision.

DESCRIPTION

     SCFFT calculates the FFT of the real vector x, and returns the
     result in the complex vector y.
     CSFFT calculates the corresponding complex-real inverse FFT.

     Consider two arrays X and Y dimensioned as follows:

	  REAL (KIND = 8), DIMENSION (0: n-1) :: X
	  COMPLEX (KIND = 8), DIMENSION (0: n / 2) :: Y

     The result Y represents the FFT of the vector X, if we apply the
     following formula:

			 n-1
	  Y (k) = scale * Sum [X (j) * w ** (isign * j * k)]; k = 0, ..., n / 2
			 j = 0


	  or,
	  w = exp (2 * pi * i / n),
	  i = + sqrt (-1),
	  pi = 3.14159 ...,
	  isign = +1 or -1.

     In general, if we calculate an FFT using SCFFT with values
     particular of isign and scale, then its inverse is calculated by
     calling CSFFT with the values ​​-isign and 1 / (n * scale).

     In particular, with the values ​​isign = +1 and scale = 1.0 provided to
     SCFTT, the inverse FFT is calculated with CSFTT by taking isign = -1 and
     scale = 1.0 / n

ARGUMENTS

     isign Scalar of type INTEGER. (Entrance)

	     Indicates whether the table of coefficients must be initialized or whether
	     apply an FFT or its inverse.

	     If isign = 0, the subroutine initializes the array table and
	     returns its value. In this case, only the arguments isign, n and
	     table are checked and used.

     n INTEGER type scalar. (Entrance).

	     Length of the FFT (Number of elements of the array x). n> 2

             A restriction of JMFFT means that this dimension must be a
             even number.

     scale Scalar of type REAL (KIND = 8). (Entrance)

	     Scale factor. Each element of the vector y is multiplied by
	     scale once the FFT has been performed as specified in the
	     formula above.

     x SCFFT: REAL type array (KIND = 8) of dimension (0: n-1). (Entrance)
	     CSFFT: COMPLEX type array (KIND = 8) of dimension (0: n / 2). (Entrance)

	     Array containing the values ​​of the elements to transform.
             See the note below.

     y SCFFT: COMPLEX type array (KIND = 8) of dimension (0: n / 2). (exit)
	     CSFFT: REAL type array (KIND = 8) of dimension (0: n-1). (exit)

	     Array containing the transformed values ​​as output.

     table Array of type REAL (KIND = 8) of dimension (100 + 4 * n). (entry or exit)
	     
             Table containing the table of coefficients and functions
	     trigonometric.

	     If isign = 0, the subroutine initializes table (table is in
	     output only).

	     If isign = +1 or -1, table is supposed to be already initialized
	     (table is input only).

     work Array of type REAL (KIND = 8) of dimension (4 + 4 * n).

	     Work table.

     isys Scalar of type INTEGER. (Entrance)
             
	     This argument is not used. It is kept for reasons
	     compatibility with CRAY's SCILIB.  

NOTES

     In the case of CSFFT, array X is a complex array that
     must also be Hermitian.

     Since only half of the array is stored
     (the first dimension goes from 0 to n / 2 inclusive), this condition is
     translates simply by the fact that x (0) must be real and that,
     if n is even, x (n / 2) must also be real.

     In fact, CSFFT automatically forces these conditions, by setting
     to 0 the imaginary part of the terms concerned. So there is
     not to worry about it.

EXAMPLES

     Example 1 : initialize the TABLE array in order to apply
     subsequently an FFT of dimension n = 1025. In this case, only the
     ISIGN, N and TABLE arguments are used.

	  REAL (KIND = 8), DIMENSION (100 + 4 * 1025) :: TABLE
	  CALL SCFFT (0, 1025, 0.d0, DUMMY, DUMMY, TABLE, DUMMY, 0)

     Example 2 : X is a real array of dimension (0: 1024) and Y is a
     complex array of dimension (0: 512). Here we apply an FFT on X and
     we save the result in Y. It is imperative to initialize TABLE,
     as shown in Example 1 before performing the FFT.

	  REAL (KIND = 8), DIMENSION (0: 1023) :: X
	  COMPLEX (KIND = 8), DIMENSION (0: 512) :: Y
	  REAL (KIND = 8), DIMENSION (100 + 4 * 1024) :: TABLE
	  REAL (KIND = 8), DIMENSION (4 * 1024 + 4) :: WORK
	  CALL SCFFT (0, 1024, 1.d0, X, Y, TABLE, WORK, 0)
	  CALL SCFFT (1, 1024, 1.d0, X, Y, TABLE, WORK, 0)

     Example 3 : we complete example 2 to perform an inverse FFT of
     Y whose result is stored in X. The scale factor SCALE is equal to
     1/1024. It is assumed that the TABLE array has been initialized previously.

	  CALL CSFFT (-1, 1024, 1.d0 / 1024.d0, Y, X, TABLE, WORK, 0)

     Example 4 : here we carry out the same calculation as in example 2. The only
     difference is in the value of the lower indices of the X arrays
     and Y which start at 1 instead of 0.

	  REAL (KIND = 8), DIMENSION (1024) :: X
	  COMPLEX (KIND = 8), DIMENSION (513) :: Y
	  CALL SCFFT (0, 1024, 1.0, X, Y, TABLE, WORK, 0)
	  CALL SCFFT (1, 1024, 1.0, X, Y, TABLE, WORK, 0)

     Example 5 : same as example 4. However, to save
     memory, the result is saved in the vector X (we assume
     that the TABLE array has been initialized previously). Must therefore
     oversize X so that it can contain Y.

	  REAL (KIND = 8), DIMENSION (1026) :: X
	  CALL SCFFT (1, 1024, 1.d0, X, X, TABLE, WORK, 0)
