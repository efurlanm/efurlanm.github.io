LAST NAME

     CCFFT - Applies a complex-complex Fast Fourier Transform (FFT).

SYNTAX

     CALL CCFFT (isign, n, scale, x, y, table, work, isys)

IMPLEMENTATION

     This subroutine emulates the subroutine of the same name of the SCILIB of
     CRAY. All real or complex arguments must be declared
     in double precision.

DESCRIPTION

     CCFFT calculates the FFT of the complex vector x, and returns the
     result in the complex vector y.

     Consider two arrays X and Y dimensioned as follows:

	  COMPLEX (KIND = 8), DIMENSION (0: N-1) :: X, Y

     The result Y represents the FFT of the vector X, if we apply the
     following formula:

			 n-1
	  Y (k) = scale * Sum [X (j) * w ** (isign * j * k)] ; k = 0, ..., n-1
			 j = 0

	  or

	  w = exp (2 * pi * i / n),
	  i = + sqrt (-1),
	  pi = 3.14159 ...,
	  isign = +1 or -1

     In general, if an FFT is applied with particular values ​​of
     isign and scale, then its inverse is calculated with the values ​​-isign
     and 1 / (n * scale).

     In particular, with the values ​​isign = +1 and scale = 1.0, the FFT
     inverse is calculated by taking isign = -1 and scale = 1.0 / n

ARGUMENTS

     isign Scalar of type INTEGER. (input)

	     Indicates whether the table of coefficients must be initialized or whether
	     apply an FFT or its inverse.

	     If isign = 0, the subroutine initializes the array table and
	     returns its value. In this case, only the arguments isign, n and
	     table are checked and used.

	     If isign = +1 or -1, the FFT or its inverse is applied.

     n INTEGER type scalar. (input)

	     Length of the FFT (Number of elements of the array x). n > = 2.

     scale Scalar of type REAL (KIND = 8). (input)

	     Scale factor. Each element of the vector y is multiplied by
	     scale once the FFT has been performed as specified in the
	     formula above.

     x Table of the COMPLEX type (KIND = 8) of dimension
             (0: n-1). (input)

	     Array containing the values ​​of the elements to transform.

     y Table of the COMPLEX type (KIND = 8) of dimension (0: n-1). (output).

	     Array containing the transformed values ​​as output. We can
             use the input array x.

     table Table of type REAL (KIND = 8) of dimension 100 + 8 * n. (entry or
             exit).

	     Table containing the table of coefficients and functions
	     trigonometric.

	     If isign = 0, the subroutine initializes table (table is in
	     output only).

	     If isign = +1 or -1, table is supposed to be already initialized
	     (table is input only).

     work Array of type REAL (KIND = 8) of dimension 8 * n.

	     Work table.

     isys Scalar of type INTEGER. (input)
             
	     This argument is not used. It is kept for reasons
	     compatibility with CRAY's SCILIB.

EXAMPLES

     Example 1 : initialize the TABLE array in order to apply
     subsequently an FFT of dimension n = 1024. In this case, only the
     ISIGN, N and TABLE arguments are used.

	  REAL (KIND = 8), DIMENSION (100 + 8 * 1024) :: TABLE
	  CALL CCFFT (0, 1024, 0.d0, DUMMY, DUMMY, TABLE, DUMMY, 0)

     Example 2 : X and Y are complex arrays of dimension (0: 1023). Here
     we apply an FFT on X and we save the result in Y. It is
     imperative to initialize TABLE, as shown in Example 1
     before performing the FFT.

	  COMPLEX (KIND = 8), DIMENSION (0: 1023) :: X, Y
	  REAL (KIND = 8), DIMENSION (100 + 8 * 1024) :: TABLE
	  REAL (KIND = 8), DIMENSION (8 * 1024) :: WORK
	  CALL CCFFT (0, 1024, 1.d0, X, Y, TABLE, WORK, 0)
	  CALL CCFFT (1, 1024, 1.d0, X, Y, TABLE, WORK, 0)

     Example 3 : we complete example 2 to perform an inverse FFT of
     Y whose result is stored in X. The scale factor SCALE is equal to
     1/1024. It is assumed that the TABLE array has been initialized previously.

	  CALL CCFFT (-1, 1024, 1.d0 / 1024.d0, Y, X, TABLE, WORK, 0)

     Example 4 : here we carry out the same calculation as in example 2. The only
     difference is in the value of the lower indices of the X arrays
     and Y which start at 1 instead of 0.

	  COMPLEX (KIND = 8), DIMENSION (1024) :: X, Y
	  CALL CCFFT (0, 1024, 1.d0, X, Y, TABLE, WORK, 0)
	  CALL CCFFT (1, 1024, 1.d0, X, Y, TABLE, WORK, 0)

     Example 5 : same as example 4. However, to save
     memory, the result is saved in the vector X (we assume
     that the TABLE array has been initialized previously).

	  COMPLEX (KIND = 8), DIMENSION (1024) :: X
	  CALL CCFFT (1, 1024, 1.d0, X, X, TABLE, WORK, 0)
